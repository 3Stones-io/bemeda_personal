üîÑ How GitHub Integration Works

Understanding the seamless workflow between site templates and GitHub collaboration

1

Component Creation

Create new components using enforced GitHub issue templates with required fields (Component ID, Type, Domain, Title, Description). Auto-validation ensures proper structure and labeling.

2

Real-Time Synchronization

GitHub Actions automatically sync component changes to the master registry, update documentation, and deploy to GitHub Pages. Status changes in GitHub Project boards reflect immediately on site pages.

3

Collaborative Management

Team members can comment, update status, assign tasks, and track progress directly in GitHub Issues. All changes are visible on both GitHub Project boards and site component pages.

4

Consistent Representation

Every component has dual representation: a human-friendly page on the site with detailed templates and a GitHub issue for collaboration. Edit buttons on site pages link directly to corresponding GitHub issues.

  • Break down into US### user stories with USS### user story steps
  • Define acceptance criteria (AC###) for each story and step
  • Identify participant types and their specific journeys
  • Map component relationships and reuse opportunities from existing scenarios

üé® Phase 2: UX/UI Design Implementation

  • Design UX### user interface flows for each story step
  • Create M### mockups for new screens and interfaces
  • Identify reusable C### UI components vs. new component needs
  • Prototype navigation patterns and interaction flows

‚öôÔ∏è Phase 3: Technical Architecture Planning

  • Define F### platform features and capabilities required
  • Design TC### technical components and service architecture
  • Plan API integrations and data flow patterns
  • Specify database schemas and system interfaces

üß™ Phase 4: Testing & Quality Assurance

  • Create T### test cases covering all user story steps
  • Validate AC### acceptance criteria for each user story
  • Document B### bug reports during testing phases
  • Plan integration testing for component interactions
  • Establish regression testing for inherited components

üí° Efficiency Note: Each new scenario typically inherits 80-95% of components from previous scenarios, significantly reducing development time and ensuring consistency across the platform.
