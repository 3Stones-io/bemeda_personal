🏗️ Three-Layer Integration

The Phoenix integration uses a sophisticated three-layer architecture that connects USE scenarios with the existing production application.

1
API Layer

REST endpoints bridging USE scenarios with Phoenix contexts, enabling executable scenario testing against real business logic.

  • POST /api/use/scenarios/:id/execute - Execute scenarios
  • GET /api/use/scenarios/:id/status - Check scenario status
  • POST /api/use/actors/:type/setup - Setup test actors
  • Integration with 97 existing test files
  • Secure scenario execution with CSRF protection
2
Event Bridge

PubSub system connecting scenario execution with LiveView for real-time monitoring and interactive scenario debugging.

  • Real-time scenario execution monitoring
  • Live actor state updates
  • Interactive scenario debugging
  • Event streaming with Phoenix PubSub
  • Integration with existing LiveView components
3
Test Integration

Elixir tests complementing Python USE scenarios, creating comprehensive end-to-end validation of business logic.

  • Elixir USE scenario test suite
  • Python BDD scenario integration
  • ExMachina factories for test data
  • End-to-end scenario validation
  • Production-ready test patterns
