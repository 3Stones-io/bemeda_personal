🎯 Integration Strategy

1
API Integration Layer

Create REST API controllers that bridge USE scenario tests with Phoenix business logic.

# Add to router.ex scope "/api/use", BemedaPersonalWeb.USE do pipe_through :api # Scenario test endpoints post "/scenarios/:scenario_id/execute", ScenarioController, :execute get "/scenarios/:scenario_id/status", ScenarioController, :status post "/actors/:actor_type/setup", ActorController, :setup end
2
USE Test Data Seeds

Create seed data that matches USE scenario test requirements.

# priv/repo/use_seeds.exs alias BemedaPersonal.{Accounts, Companies, JobPostings} # Create Healthcare Organisation actor {:ok, healthcare_org} = Accounts.register_employer(%{ email: "admin@test-hospital.com", first_name: "Test", last_name: "Hospital" }) # Create Job Seeker actor {:ok, job_seeker} = Accounts.register_job_seeker(%{ email: "nurse@example.com", first_name: "Jane", last_name: "Nurse" })
3
Scenario Event Bridge

Use Phoenix PubSub to bridge scenario events with LiveView real-time updates.

# lib/bemeda_personal/use/scenario_bridge.ex defmodule BemedaPersonal.USE.ScenarioBridge do alias Phoenix.PubSub def broadcast_scenario_event(scenario_id, event, data) do PubSub.broadcast( BemedaPersonal.PubSub, "use_scenario:#{scenario_id}", {event, data} ) end end
4
Elixir USE Test Suite

Create Elixir tests that complement the Python USE scenarios.

# test/use_scenarios/business_scenarios_test.exs defmodule BemedaPersonal.USEScenarios.BusinessScenariosTest do use BemedaPersonal.DataCase use BemedaPersonalWeb.ConnCase describe "B_S001: Cold Call to Placement" do test "US001: Organisation Receives Cold Call", %{conn: conn} do # Given a Healthcare Organisation exists org = create_test_organisation() # When the Sales Team makes contact response = post(conn, "/api/use/scenarios/B_S001_US001/execute") # Then the organisation should be engaged assert json_response(response, 200)["status"] == "success" end end end
5
