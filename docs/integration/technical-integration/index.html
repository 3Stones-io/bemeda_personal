‚ö†Ô∏è Common Issues & Solutions

GitHub Actions Workflow Failing
Check workflow permissions in repository settings. Ensure GITHUB_TOKEN has sufficient permissions for Issues, Projects, and Pages deployment. Update workflow YAML if needed.
USE Scenarios Not Finding Phoenix Application
Verify Phoenix server is running on correct port (usually 4001 for test). Check API_CLIENT configuration in features/support/api_client.py. Enable mock mode if Phoenix unavailable.
Component Registry Not Syncing
Check if issues have proper labels ("component"). Verify GitHub Actions workflow is triggered on issue changes. Check MASTER-REGISTRY.json generation logs.
GitHub Pages Deployment Issues
Ensure repository has Pages enabled with "gh-pages" branch as source. Check deploy-pages.yml workflow status. Verify all HTML files have valid syntax.
Issue Template Not Enforcing Validation
Verify component-definition.yml is in .github/ISSUE_TEMPLATE/ directory. Check that validation workflow (auto-label-components.yml) is active and has proper permissions.
