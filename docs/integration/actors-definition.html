Human Actors

Real people who interact with the platform to achieve business goals

A1

Healthcare Organisation

Healthcare facilities (hospitals, clinics, care homes) seeking qualified healthcare professionals for permanent or temporary positions.

Primary Steps:
S2 S3 S4 S5 S6
Goals: Find and hire qualified healthcare professionals quickly and efficiently
Pain Points: Staff shortages, urgent needs, quality candidate matching
A2

Sales Team

Bemeda's sales representatives who identify healthcare organisations with staffing needs and facilitate platform adoption.

Primary Steps:
S1 S13 S14 S15 S16 S17 S18
Goals: Convert leads into active platform users, maintain client relationships
Tools: CRM system, call scripts, platform demos, success metrics
A7

JobSeeker

Healthcare professionals (nurses, doctors, specialists) looking for job opportunities that match their skills and preferences.

Primary Steps:
S7 S8 S9 S10 S11 S12
Goals: Find suitable job opportunities, streamlined application process
Preferences: Location, shift patterns, specialization, salary expectations

System Actors

Technical components that participate in scenario execution

A3

Authentication System

Core security component handling user authentication, authorization, and session management for all platform users.

Used in:
T_S1_API T_S3_API T_S3_TEST T_S7_AUTH
Technology: Phoenix Auth, JWT tokens, dual user types (employer/job_seeker)
Foundation: F1 - Shared across all scenarios
A4

Database

PostgreSQL database storing all platform data including user profiles, job postings, applications, and system state.

Used in:
T_S2_DB T_S3_DB T_S7_DATA T_S9_DB
Technology: PostgreSQL 14+, Ecto ORM, migrations, indexes
Foundation: F2 - Core data persistence layer
A5

Desktop User

Browser environment representing desktop users interacting with the platform UI through standard web browsers.

Used in:
U_S1_WIRE U_S2_WIRE U_S3_WIRE All _INTERACT
Requirements: Modern browsers (Chrome, Firefox, Safari, Edge), 1280px+ viewport
UX Focus: Full-featured interface, keyboard navigation, multi-column layouts
A6

Accessibility Tool

Automated and manual accessibility testing tools ensuring WCAG 2.1 AA compliance across all user interfaces.

Used in:
U_S2_ACCESS U_S3_ACCESS All UX validation
Tools: axe DevTools, WAVE, screen readers (NVDA, JAWS), keyboard testing
Standards: WCAG 2.1 AA, ARIA best practices, semantic HTML
